# Incident: 2026-04-07 09-53-04

## Summary

```md
Between the hours of 9:53 AM and 12:07 PM on 2026-04-07, all users encountered the inability to purchase or verify JWT pizzas. The event was triggered by a planned chaos test at a random time between 8 AM and 3 PM. The test caused the outage of the pizza factory which JWT pizza relies on.

The outage of this factory caused the JWT pizza system to be unable to get pizzas or verifications from it. The alerting system failed to alert the engineers of this outage, which led to a longer outage than expected. The team started working on the event by reviewing the incoming logs. This critical incident affected 100% of users.
```

## Detection

```md
This incident was detected when the engineer reviewed the metrics and logs of the application.

The alerts were not setup properly which led to delaying the response by 2 hours.

Proper alerts on all core systems will be set up by Jonah so that all outages to core systems will be detected immediately in the future.
```

## Impact

```md
For 2 hours and 14 minutes between 9:53 AM and 12:07 PM on 04/07/26, 100% our users experienced this incident.

This incident affected 1 customer 100% of JWT pizza users, who experienced no way to order or verify JWT pizzas.

No support tickets or social media posts were submitted.
```

## Response

```md
After discovering the outage at 12:00 PM, Jonah came online at 12:00 PM in the grafana JWT pizza dashboard.

Jonah detected the issue at 12:07 PM and fixed the issue.
```

## Root cause

```md
The application does not have anything to fall back on when the factory fails, such as a way to store orders and fulfill them later.
```

## Resolution

```md
The service was restored by following a link from the test that returned functionality. This was primarily designed to test the ability of engineers to respond. I knew what steps to take because I had setup the metrics and logging systems so that I could see where issues were and I was able to isolate what part of the system was failing based on what metrics were having issues. The logging system allowed me to see what was happening in communication between the server and the factory at the time when the metrics for pizza orders, revenue, and factory latency all went to zero.
```

## Prevention

```md
In order to fix future issues with this I am going to setup more robust alerts. One way that this issue could be prevented in the future is to make it so that if the application does not get a response from the factory, to tell the user and then begin storing orders to be fulfilled in the future, telling the customers that we will follow up with them before making the purchase to confirm.
```
